# AGENTS.md

本文件是给 Codex / Claude / Cursor 等 AI 编程助手使用的项目协作规则。处理本仓库任务时，必须优先遵守这里的约定。

## 核心原则

- 不要做兜底式、掩盖式、临时绕过式修复。
- 遇到问题要定位根因，一次性把相关问题修完整。
- 不要为了让页面“看起来能跑”而保留错误状态、重复逻辑或无效分支。
- 如果发现改动会影响多个入口、组件或构建产物，要一次性同步处理。

## 修改要求

- 修改源码、配置、静态资源或页面入口后，必须同步更新 `dist/`。
- 本项目使用 Vite 构建，常规同步方式是运行：

```bash
npm run build
```

- 不要只修改 `dist/` 而不修改对应源文件，除非任务明确要求直接处理构建产物。
- 如果修改了 `src/`、`public/`、根目录 HTML 入口、`vite.config.js` 或依赖配置，完成后都要重新构建。
- 修改时要尊重已有工作区状态，不要回滚或覆盖与当前任务无关的用户改动。

## 质量与效率要求

- 为了节约时间，默认只保证代码构建无报错。
- 具体实现效果不要主动运行 Playwright 或做浏览器自动化检查，除非用户明确要求。
- 页面视觉效果、交互细节和最终体验由用户自行查看与确认。
- 保证代码具备较强复用性，避免重复实现和一次性写法。
- 保证代码具备较强可维护性，命名清晰，结构合理，逻辑边界明确。
- 保证软件性能优化良好，避免不必要的重复渲染、重复计算、过大资源和低效流程。

## 验证要求

- 每次代码修改后至少运行：

```bash
npm run build
```

- 构建完成后检查 `git status --short`，确认源码变更和 `dist/` 变更都已纳入工作区。
- 不要主动运行 Playwright 或浏览器自动化验证，除非用户明确要求。
- 如用户要求检查页面效果，再按用户指定范围进行预览、截图或 Playwright 验证。

## 提交与发布

- 每次完成修改后，应提交变更并推送到 GitHub 仓库：

```bash
git add .
git commit -m "描述本次修改"
git push origin main
```

- 推送后要确认 GitHub Pages / GitHub.io 已更新。
- 当前项目的 GitHub Pages 路径通常对应：

```text
https://1ts0k.github.io/live2love/
```

- 如果 GitHub Pages 没有自动更新，应根据仓库的 Pages 配置继续完成发布流程，直到线上页面更新。

- 预览命令：`npm run preview`
